# Story 3.3: Post CRUD API 구현 (2/2 - 생성/수정/삭제)

## 메타 정보
- **Epic**: Epic 3 - 백엔드 API 구현
- **Priority**: High
- **Estimate**: 1.5일 (12시간)
- **Dependencies**: 3.2 완료

---

## Description

Post 생성/수정/삭제 API 구현 (관리자 전용)

### 작업 내용
- POST /posts (생성, AdminGuard)
- PATCH /posts/:id (수정, AdminGuard)
- DELETE /posts/:id (삭제, AdminGuard)
- slug 자동 생성 (한글 → 영문)

---

## Acceptance Criteria

- [ ] POST /posts (title, content, categoryId, isPublic) → 글 생성
- [ ] slug 자동 생성 (예: "리액트-훅스" → "리액트-훅스")
- [ ] PATCH /posts/:id → 글 수정
- [ ] DELETE /posts/:id → 물리적 삭제
- [ ] 관리자만 생성/수정/삭제 가능
